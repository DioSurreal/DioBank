we will breakdown user support like:
	metrics:
		money transfer : 
			Internet Banking(IB):
				no of agreemants ≈ 40M
				transactions per mounth ≈ 100M
			Mobile Banking(MB) :
				no of agreemants ≈ 150M
				transactions per mounth ≈ 40B	
			IB + MB : (100M + 40B) / (30*24*60*60) 
				: ≈15,818 TPS > 20,000 TPS (avg)
			for peek: x10 transactions ≈ 200,000 TPS
		kyc images 	 :
			no of image  IB + MB or 40M + 150M ≈ 190M images
			size of storage  190M * 3 * 50kB ≈ 28.5 TB *3(As a precaution) ≈	86TB
		Cheque : 
			transactions per mounth ≈ 200,000 transaction/mounth
		Loan	:
			transactions per mounth ≈ 200,000 transaction/mounth

	componect user journey:
		
		money transfer :
			(app,web) -> API gateway(auth,jwt) -> check balance(Spring Boot +Kafka Streams + RocksDB ) ->(Kafka partition) -> double ledger entry(kafka topic)
		kyc (first open account) :
			(app,counter) -> fill identified data -> camera shot (portatait,front identify card, back identify card) -> Upload Image ──► [ Object Storage (S3/MinIO) ]
                                                															│
                                      															(Returns Image URL String)
                                               																 │
                                      															                 ▼
														[User Service] ───────────────────────► Save URL to UserDB

		cheque service:
			cheque transactions -> validate cheque -> clearing cheque(batch processing)
		loan service :
			loan transactions -> fraud service -> loan approved -> calculate loan model (installment per mounth) -> loan contract
	high level design :
		
		user service :
			Service Communication Layer : 
						gRPC :   insert_user_data : protobuf(user_data) -> user_id	
						         get_uder_data -> user_id -> protobuf(user_data)
						         update_user_data user_id,field -> user_id 
		ledger service : 
			Service Communication Layer :
						kafka : Consumer insert_entry_double_ledger : 2 ledger entry -> ledger_id
		account service :
			Service Communication Layer :
						gRPC :   insert_account_data : protobuf(account_data) -> account_id	
						         get_uder_data -> account_id -> protobuf(account_data)
						         update_account_data account_id,field -> account_id 							   
		cheque service :
			Service Communication Layer :
						gRPC :  insert_cheque_transaction : cheque_data -> cheque_id
						validate cheque : cheque_data -> VALIDATED_SUCCESS or INVALID_SIGNATURE/FORMAT (Rejected).
						batch peocess : clearing house  -> ACCEPTED_FOR_CLEARING, not CLEARED
		loan service :
   			Service Communication Layer :
        				gRPC : CreateLoanTransaction : protobuf(loan_data) -> loan_id    
               				   GetLoanTransaction    : loan_id -> protobuf(loan_data)
               				   GetLoanApprovalStatus : loan_id -> status(APPROVED, REJECTED, or PENDING_REVIEW)
        				Internal Engine : 
               				   Loan Calculator Core  : Calculates installment, fee, interest, policy maturity dates, etc.
		BFF :
			separate for communicate with each service like :
					Mobile-Banking-BFF , Web-Banking-BFF , cheque BFF , loan BFF
